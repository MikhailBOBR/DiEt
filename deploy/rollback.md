# Rollback runbook

1. Остановить дальнейшие production deploy через GitHub environment.
2. Сохранить логи неуспешного deploy, commit SHA и результат smoke-test.
3. Вернуть предыдущий успешный deploy средствами Render или повторно развернуть предыдущий commit.
4. Проверить `/api/live`, `/api/ready`, `/api/health` и базовые операции API.
5. Не выполнять автоматический откат схемы БД: текущие миграции рассчитаны на forward-compatible изменения.
6. Если изменение схемы несовместимо, развернуть исправляющую forward migration. Восстановление backup допускается только по отдельному решению с оценкой потери данных.
7. Зафиксировать результат в deployment evidence и описать причину инцидента.
